2. Mt s khi nim cn nhc li

Pixel thit b:

L 1 im nh, n ph thuc vo kch thc mn hnh, v phn gii mn hnh. Thng thng, 1px thit b trn cc mn hnh u c gi tr khc nhau.

Pixel ny l pixel vt l ca thit b khc hon ton so vi n v px m ta hay s dng trong CSS.

phn gii thit b - Resolution:

Cc bn thng thy mn hnh c phn gii 4k, HD, FullHD, km theo n l 2 con s c dng P1xP2 phi khng no?

Nhng con s ny th hin s pixel c trn mn hnh, P1 l s pixel phn b theo chiu di, cn P2 l theo chiu rng.

DPI, PPI:

DPI - Dots Per Inch. PPI - Points Per Inch

Chng u l s pixel/im nh c trn 1 inch di vt l ca thit b, biu th mt pixel trn mn hnh.

3. Density Independent Pixels

3.1 t vn

Cc in thoi Android c nhiu loi kch thc (phone, tablet, TV, ), hn na chng li cn c phn gii khc nhau, dn n kch thc pixel lun thay i, gy kh khn cho nh pht trin khi dng giao din.

Ly v d n gin nht v 2 in thoi c cng kch thc ng cho 5inch, nhng 1 chic FullHD 1080x1920, 1 chic 4K 2160 x 3840.

Nu bn l dev, bn s mun 1 button trong app ca mnh hin th trn 2 in thoi ny ging nhau phi khng no?

Th nhng nu lm vic vi px thit b, ta s gp 1 cht rc ri v vi cng 1 kch thc, in thoi 4K lun cn gp i s px so vi FullHD.

2 ci mt, trn thc t, ta c ty t loi mn hnh, vy th style sao cho xu, cha k cn n ti u cc file nh cho app.

Chnh v vy, dp ra i.

3.2. DP

Google nh ngha n nh sau:

One dp is a virtual pixel unit thats roughly equal to one pixel on a medium-density screen (160dpi; the baseline density). Android translates this value to the appropriate number of real pixels for each other density.

Nu bn c bi trc ca mnh v CSS pixel, th c th thy n kh quen thuc phi khng no, y cng l 1 virtual unit.

1dp tng ng 1 px trn mn hnh 160dpi. C ngha l 1dp = 1 / 160 = 0.00625 (inch) = 1px (160dpi)

quy i dp sang px trn cc loi mn hnh khc, ta nhn dp vi 1 s, gi l scale factor (t l DPI ca mn hnh so vi chun 160dpi).

V d my 320dpi. 1dp = 1 * 320 / 160 = 2px

Android s t ng convert sang pixel thit b cho ta, nn ta ch cn dng dp l trn mi thit b, phn t ca ta s u c kch thc bng nhau (ngoi l s cp sau).

Khi ta dng UI, nn tn dng ti a dp, trnh s dng px.

Bn cnh , Google cng khuyn ta khng nn s dng dp cho text, m hy dng sp. y cng l mt loi density-independent, nhng n c th b nh hng bi ty chn ca ngi dng. Ngi dng chn c ch to, th 1sp cng ln ln theo. V vy, cng trnh dng sp khi thit k layout.

3.3 Density Bucket

dp ra i, nhng cng cha gii quyt c ht cc vn . chnh l v resource hnh nh. Style cho nh th c th s dng dp c (nh s b ko to, hoc co li cho size cn thit), nhng resource th u c th to nh ty c, thng s ca n lun l c nh.

Ly v d v 1 hnh nh 500x500px (resolution), vi 2 mn hnh c kch thc bng nhau, nhng khc phn gii (khc DPI), 1 chic l 360x640, 1 chic l 720x1280.

Hnh nh trn nu hin th trn mn hnh 360x640 th chng phi s d tha sao? V chiu rng mn hnh ch c 360px. Cn trn mn hnh 720x1280, th n ch ng l bao. V vy ta cn s dng hnh nh c phn gii thp hn cho mn hnh 360x640.

Nhng khng l c mi phn gii li to 1 hnh nh tng ng? Vy th app size ca ta s bng n lun.

V l m ra i density bucket.

cho n gin, cc loi mn hnh c DPI gn bng nhau s c lm trn ri gom vo 1 nhm. V d nh my 293dpi, Android s coi nh my l 320dpi, khi 1dp = 2px. Cn nhiu loi khc na, nhng mnh ch nu ra 4 bucket ng ch , best practice l support c 4 bucket ny.
