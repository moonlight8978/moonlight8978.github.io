3. Range optimization

  • range l mt access method
  • range s dng index ly ra nhng subset ca kt qu cui cng.
  • range support c single-part ln multiple-part index.

Nhng trng hp sau, MySQL s coi nh l range condition:

  • Mnh WHERE i vi nhng trng c nh index (s dng BTree hoc Hash), m s dng nhng ton t sau: =, <=>, IN(), IS NULL, hay IS NOT NULL

    i vi, BTree, ngoi nhng php ton trn, cn support thm >, <, <=, >=, BETWEEN, !=, hay <> vi constant value, hoc LIKE cng l range condition. Ch khi s dng LIKE, v phi phi l 1 string hng, v khng bt u bi wildcard nh % hay _.

  • Kt hp nhiu range condition s dng AND hoc OR, ta vn thu c range condition.

Constant value

L gi tr c tnh ton trc thi im runtime, khi runtime, gi tr ca n s khng thay i
  • Hng s truyn thng vo tham s ca cu truy vn
  • Mt ct ca const hay system table
    • const table l bng ch c ti a 1 row (hay 0 hoc 1 row).
    • const table c th l kt qu ca 1 cu truy vn cha mnh WHERE i vi 1 field unique, not null, c dng column = constant. Truy vn ny lun tr v 1 kt qu duy nht.
    • table c 1 row th gi l system table

BTree, Hash index

BTree:

  • Self-balancing tree, cu trc dng cy, c th t cn bng nhm gi chiu cao ca cy thp nht c th.

  • Trnh nhm vi Binary Tree (cy nh phn)

  • Thi gian tm kim O(logn)

  • Ph hp vi a dng cc php ton: =, <=>, IN(), IS NULL, IS NOT NULL, >, <, <=, >=, BETWEEN, !=, <>, hay thm ch c LIKE

Hash:

  • Bng bm, l cu trc d liu lu theo key-value

  • Tm kim theo key rt nhanh - O(1)

  • Ph hp vi nhng php ton: =, <=>, IN(), IS NULL, hay IS NOT NULL.

3.1. Single-part index

Single-part index l nhng index c nh ring l cho 1 field.

Khi thc hin truy vn vi nhng index loi ny, vi mi possible key (c th s dng EXPLAIN check), MySQL s tin hnh extract range condition. Nhng iu kin khng th cu thnh range condition s b loi b, nhng iu kin c th b overlap s c gp vi nhau.

Sau khi extract, MySQL s p dng nhng iu kin tn dng c ti a index ca table, sau kt hp thm vi nhng iu kin cn li lc tip.

MySQL ch b qua index, nu nh n tin rng vic duyt full table ti u hn, hoc dng FORCE INDEX.

Index Dive

Index dive c thc hin trong khi optimizing tnh ton estimate (s row tha mn 1 condition), quyt nh xem c nn dng index hay khng. C th skip bng cch dng FORCE INDEX

Nu iu kin phc tp th index dive s mt nhiu thi gian.

Ngoi index dive, MySQL cng c th s dng index statistics estimate, tuy nhin chnh xc thp hn, c th dng ANALYZE TABLE update li index statistics, tng cng chnh xc.

Index dive b skip trong trng hp tha mn tt c iu kin sau (ch p dng cho single table query):

  • FORCE INDEX
  • Nonunique index, v khng phi FULLTEXT index.
  • Khng c subquery
  • Khng c DISTINCT, GROUP BY, hoc ORDER BY.
Equality Range Optimization of Many-Valued Comparisons

Nhng biu thc nh trn, vi col_name c nh index, v so snh vi nhiu gi tr, c gi l range comparisions (mi range l 1 gi tr). Optimizer s estimate cost ca mi range nh sau:

  • Nu index l unique, cost = 1
  • Nu khng unique, s cn s dng index dive hoc index statistics estimate.

ng vi mi range l 2 ln dive (1 cho im bt u, 1 cho im kt thc) ca range (interval).
